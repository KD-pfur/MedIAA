# MedFlow MVP

Демо-MVP для разбора медицинских документов с GigaChat OCR и AI-чатом по документу.

## Что умеет

- загрузка `.txt`, `.md`, `.csv`, `.json`
- загрузка изображений `.png`, `.jpg`, `.jpeg`, `.webp`
- OCR изображений через GigaChat API на сервере
- предпросмотр исходной и улучшенной картинки перед OCR
- серверная предобработка изображения: автоповорот, crop, усиление контраста, бинаризация
- режим `Форма или бланк` для медсправок и документов с полями
- отдельные OCR-слои для печатного текста и синей рукописи
- извлечение:
  - пациента
  - даты документа
  - врача и клиники
  - диагнозов и жалоб
  - назначений
  - лабораторных показателей с оценкой отклонений
- извлечение полей формы: ФИО, дата рождения, адрес, место учебы/работы, заболевания, прививки
- диалоговое окно для вопросов по текущему кейсу
- сохранение кейсов в `localStorage`
- экспорт результата в JSON

## Как запустить

1. Создайте локальный файл `.env` на основе примера:

```bash
cp .env.example .env
```

2. Откройте `.env` и вставьте свой ключ GigaChat:

```env
GIGACHAT_AUTH_KEY=your_gigachat_auth_key_here
GIGACHAT_SCOPE=GIGACHAT_API_PERS
GIGACHAT_MODEL=GigaChat-Max
GIGACHAT_CA_BUNDLE=
```

3. Запустите сервер из директории проекта:

```bash
python3 server.py
```

Потом откройте:

```text
http://localhost:8000
```

## Ограничения

- Это demo-MVP, не медицинская система и не клиническая рекомендация.
- PDF по-прежнему не разбираются напрямую, для них нужен OCR-текст или предварительная конвертация в изображение.
- Разбор построен на эвристиках и шаблонах, а не на сертифицированной медицинской логике.
- AI-чат использует GigaChat API через локальный Python proxy.
- OCR по кнопке `Распознать текст с картинки` работает только через GigaChat API.
- Если `GIGACHAT_AUTH_KEY` не задан в `.env` или окружении, OCR по картинкам работать не будет.
- Для TLS-соединения с GigaChat может понадобиться сертификат Минцифры; путь можно указать в `GIGACHAT_CA_BUNDLE`.
